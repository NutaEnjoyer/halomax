Дизайн представляет собой чистую, минималистичную и ориентированную на данные панель управления (dashboard). Основной упор сделан на ясность, читаемость и быструю визуальную оценку ключевых метрик и статусов. Стиль современный, с большим количеством "воздуха" (свободного пространства), что делает интерфейс не перегруженным.

1. Структура и Макет (Layout)
Основной макет: Двухколоночный.

Левая колонка (Сайдбар): Фиксированная по ширине, содержит основную навигацию по разделам. Всегда видна на десктопных устройствах.

Правая колонка (Основной контент): Занимает оставшееся пространство. Содержит заголовок текущей страницы и контентные блоки (карточки, таблицы, графики). Имеет собственную вертикальную прокрутку.

Шапка (Header): Расположена вверху, растянута на всю ширину страницы. Содержит логотип, название системы, а справа — информацию о пользователе и кнопку выхода.

2. Цветовая палитра
Фон:

Страница: Очень светлый серый (например, #F5F6FA).

Контентные блоки (карточки, таблицы): Чистый белый (#FFFFFF).

Основной акцентный цвет (Brand Blue): Яркий, насыщенный синий (похож на #4A90E2 или #0D6EFD). Он используется для:

Логотипа.

Кнопок основного действия ("Обновить", "Экспорт в Excel").

Выделения активного пункта в меню навигации (фон).

Ссылок и интерактивных элементов.

Основного цвета в диаграммах (например, в круговой диаграмме).

Текст:

Заголовки: Темно-серый или почти черный (#212529).

Основной текст: Серый (#495057).

Второстепенный текст (подписи, даты): Светло-серый (#6C757D).

Семантические цвета (для статусов, рисков, уведомлений):

Проблема/Ошибка (Красный/Розовый): Используется для обозначения проблем, просрочек, высокого риска. Фон — светло-розовый, текст — темно-красный.

Предупреждение (Желтый/Оранжевый): Для умеренных рисков или показателей, требующих внимания. Фон — светло-желтый, текст — темно-желтый/оранжевый.

Успех (Зеленый): Для завершенных задач, доставки вовремя, низкого риска. Фон — светло-зеленый, текст — темно-зеленый.

Информация (Синий/Голубой): Для нейтральных статусов, таких как "в пути". Фон — светло-голубой, текст — синий.

3. Типографика
Шрифт: Используется современный шрифт без засечек (sans-serif), например, Inter, Roboto, или системный шрифт.

Иерархия текста четко выражена:

H1 (Заголовок страницы, "Отправления"): Крупный размер (28-32px), жирное начертание (SemiBold/600).

H2 (Заголовок карточки, "Обнаруженные проблемы"): Средний размер (20-22px), среднее начертание (Medium/500).

KPI / Крупные цифры на дашборде: Очень крупный размер (36-40px), жирное начертание (Bold/700).

Текст в таблицах и карточках: Стандартный размер (14px), обычное начертание (Regular/400).

Заголовки таблицы: Стандартный размер (14px), но с жирным начертанием (SemiBold/600).

4. UI Компоненты (Элементы интерфейса)
Сайдбар (Navigation):

Каждый пункт состоит из иконки и текстовой метки.

Активный пункт выделяется синим фоном и белым цветом текста/иконки. Имеет скругленные углы (border-radius: ~8px).

Карточки (Cards):

Имеют белый фон, скругленные углы (border-radius: ~12px) и легкую тень (box-shadow), чтобы "приподнять" их над фоном страницы.

Внутри карточек достаточно свободного пространства (внутренние отступы padding: ~24px).

Кнопки (Buttons):

Основные (Primary): Сплошная синяя заливка, белый текст, скругленные углы (~8px). Часто содержат иконку.

Вторичные/Фильтры (Tabs): Группа кнопок ("Все (40)", "В пути (27)"). Активная кнопка имеет стиль основной, неактивные — светло-серый фон и темный текст.

Таблица (Table):

Минималистичная: нет вертикальных линий, только горизонтальные разделители между строками.

Строки не чередуются по цвету, что создает чистый вид.

Заголовки выделены жирным шрифтом.

Теги/Плашки статусов (Pills/Badges):

Это ключевой элемент для быстрой оценки. Статусы ("Покинуло сортировочный центр", "8 дней") оформлены в виде цветных "плашек" со скругленными углами и фоном, соответствующим семантическому цвету.

Поля ввода (Input):

Стандартное поле с тонкой серой рамкой, скругленными углами (~8px) и иконкой поиска слева.

Диаграммы (Charts):

Простые и наглядные, без лишних деталей. Круговая диаграмма использует основные и семантические цвета. Графики (как "Динамика доставок") — простые линейные или областные.

Ключевые моменты для реализации:

Консистентность: Соблюдайте одинаковые отступы, скругления углов и размеры шрифтов для однотипных элементов по всему приложению.

Адаптивность: Предусмотрите, как интерфейс будет выглядеть на мобильных устройствах. Скорее всего, сайдбар будет скрываться в "бургер"-меню, а карточки и колонки таблицы выстроятся вертикально.

Интерактивность: Добавьте hover-эффекты (эффекты при наведении курсора) для всех интерактивных элементов: кнопок, ссылок, строк таблицы (например, легкое изменение фона).

Иконки: Используйте единый набор контурных (line style) иконок, например, Feather Icons, Heroicons или Font Awesome.

Этот подход позволит создать визуально приятный, функциональный и масштабируемый интерфейс, который будет интуитивно понятен пользователям.
